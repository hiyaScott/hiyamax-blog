一、协作机制总览

┌─────────────┐ ┌─────────────┐ ┌─────────────┐ │ 飞书讨论 │────▶│ AI平台执行 │────▶│ Web呈现 │ │ (指令下达) │ │ (Token消耗) │ │ (成果展示) │ └─────────────┘ └─────────────┘ └─────────────┘ ▲ │ └──────────────────────────────────────────┘ (反馈确认)

核心原则：飞书对话驱动 → Pipeline页面记录 → Web成果展示 → 用户确认后再推进

二、详细流程说明（7个Step）

阶段 飞书指令 AI执行 呈现形式 成本
1 剧本创作 "开始讨论剧本" 整理故事结构、人物关系 Pipeline页面文字 ¥0
2 角色设计 "开始角色设计" 即梦AI生成角色图 角色参考图 + 三视图 即梦Token
3 场景设计 "开始场景设计" 即梦AI生成场景概念图 场景参考图 即梦Token
4 分镜首尾帧 "生成首帧/尾帧" 即梦/Seedream多图参考生成 首帧图 + 尾帧图 + 历史版本 即梦/Seedream Token
5 视频生成 "生成视频" 海螺/可灵API首尾帧生成 视频片段 + 成本记录 API按次计费
6 后期剪辑 "开始后期" FFmpeg处理 成片视频 ¥0
7 发布 "部署" GitHub Push 专题页面URL ¥0

各阶段详细说明

Step 1: 剧本创作

触发：用户在飞书发送项目启动指令

内容：讨论故事核心、角色、情感基调、时长、结构、关键记忆点

输出：更新 pipeline/data/{project}.json，包含故事结构、角色设定、关键记忆点、视觉风格

Step 2: 角色设计

触发：用户说"开始角色设计"

执行：调用即梦AI生成角色参考图（如有手绘则图生图转3D写实风格）

输出：角色参考图、三视图合成图保存到 assets/{project}/，Pipeline页面展示手绘→AI对比

Step 3: 场景设计

触发：用户说"开始场景设计"

执行：根据剧本描述，使用即梦AI生成关键场景概念图

输出：场景参考图保存到 assets/{project}/scenes/，用于分镜生成时参考环境氛围

Step 4: 分镜首尾帧

触发：用户说"生成首帧/尾帧"或"开始分镜"

执行：使用即梦AI或Seedream 5.0，基于角色三视图+场景图生成每个镜头的首帧和尾帧

关键技术：

  • 多图参考：角色三视图 + 场景图 + 姿态参考
  • Seedream 5.0支持base64直接传图，避免URL下载超时
  • 首尾帧应为同一"决定性瞬间"的不同镜头构图
  • 保留历史版本(frameHistory数组)，便于回溯对比

输出：首帧图 + 尾帧图 + 缩略图 + 生成参数记录

Step 5: 视频生成

⚠️ 重要资金保护机制

  • 调用前检查账户余额
  • 单次任务最多3次重试
  • 用户明确说"开始"后才执行
  • 用户说"停"必须立即停止
  • 区分"提交任务"和"查询状态"的不同API端点

API选择：

  • 海螺AI (MiniMax-Hailuo-02)：首尾帧模式，6秒，适合镜头运动
  • 可灵AI (kling-v2-6)：图生视频，5/10秒可选，角色稳定性好

输出：MP4视频片段 + 任务ID + 生成成本记录

Step 6: 后期剪辑

工具：FFmpeg（本地处理，无额外成本）

流程：合并片段 → 添加硬字幕（Noto Serif CJK字体）→ 生成BGM → 混音输出

Step 7: 发布

部署：GitHub Pages

输出：专题页面URL + Pipeline页面更新 + 合集页面入口

三、指令类型与响应标准

🚀 启动指令
"开始做AI短片"
确认需求 → 开始Step 1讨论
✅ 确认指令
"确认这个方案"
记录决策 → 推进到下一步
⚡ 执行指令
"生成角色图"
检查前提 → 执行 → 汇报结果
⏸️ 暂停指令
"先停一下"
立即停止 → 等待指示
🔄 修改指令
"重新生成第3个"
执行修改 → 更新Pipeline
📤 部署指令
"部署这个视频"
执行发布 → 提供URL

四、必须遵守的规则

  • 1. 用户说"停"必须立即停止 无论当前进行到哪一步，不要擅自继续
  • 2. 遇到困难先问，不自己乱试 API调用失败时询问用户，不要反复重试导致资金浪费
  • 3. 资金保护机制 API调用前检查余额；单次任务最多3次重试；高频调用加30秒间隔
  • 4. 阶段性确认 每完成一个Step必须用户确认，不要擅自进入下一阶段

七、故障处理流程

API调用失控（如第147天事故）

  1. 立即停止脚本
  2. 记录事故：调用次数、损失金额、时间范围
  3. 通知用户：汇报事故详情
  4. 修复脚本：区分提交/查询端点，加频率限制
  5. 手动补救：用户从后台获取视频链接，逐个下载
  6. 更新MEMORY.md：记录教训和防范措施

字幕乱码

  1. 问题定位：检查字体是否支持中文
  2. 解决方案：改用Noto Serif CJK等CJK字体
  3. 重新压制：FFmpeg重新生成视频
  4. 验证：确认中文正常显示
